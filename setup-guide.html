Production Setup Guide

Deploy WhiskyPriceHub on any VPS in minutes. Uploads a Spiritory CSV, scrapes WhiskyBase prices, and streams live progress to the browser.

dnsUbuntu 22.04+
deployed_codeDocker 24+
languagePort 3000 + 8000
Before you start
  • check_circle VPS running Ubuntu 22.04 or later — 2 vCPU / 2 GB RAM minimum (Chromium needs headroom)
  • check_circle Ports 3000 (frontend) and 8000 (backend) open in your firewall / security group
  • check_circle A domain pointed at the server (optional but recommended for HTTPS via reverse proxy)
  • check_circle SSH access to the server as root or a sudo user
Installation — 5 steps
1
Install Docker
One command — installs Docker Engine and the Compose plugin
terminal — install Docker 
curl -fsSL https://get.docker.com | sh
Verify with docker --version and docker compose version before continuing.
2
Clone the repository
Fetch the code and enter the project directory
terminal 
git clone https://github.com/AlejandroTechFlap/WhiskyPriceHub.git
cd WhiskyPriceHub
3
Configure environment variables
Copy the template and fill in your domain and password
warning .env contains secrets — never commit it to Git. It is already in .gitignore.
step 3a — create .env 
cp .env.example .env
nano .env
VariableTypeDescription
NEXT_PUBLIC_API_URL Required Public URL of the backend — used by the browser for SSE. E.g. https://api.yourdomain.com or http://YOUR_IP:8000
ALLOWED_ORIGINS Required Frontend origin allowed by CORS. E.g. https://yourdomain.com or http://YOUR_IP:3000
AUTH_USERNAME Required Login username for the frontend. Default: Spiritory
AUTH_PASSWORD Required Login password for the frontend — choose a strong password
PROXY_URL Optional Residential proxy to reduce Cloudflare bans. Format: http://user:pass@ip:port

Full .env template to paste:

.env — full template 
NEXT_PUBLIC_API_URL=https://api.yourdomain.com
ALLOWED_ORIGINS=https://yourdomain.com
AUTH_USERNAME=Spiritory
AUTH_PASSWORD=change_me
# PROXY_URL=http://user:pass@ip:port
info No domain yet? Use http://YOUR_SERVER_IP:8000 for NEXT_PUBLIC_API_URL and http://YOUR_SERVER_IP:3000 for ALLOWED_ORIGINS.
4
Create the data directory
Persistent volume for SQLite database and uploaded CSV files
terminal 
mkdir -p data
5
Build and start
Builds both containers and starts them in the background
terminal — first run (~5 min, installs Chromium) 
docker compose up -d --build
check_circle Frontend: http://YOUR_SERVER_IP:3000 — login with your AUTH_USERNAME / AUTH_PASSWORD
Backend: http://YOUR_SERVER_IP:8000/docs — FastAPI interactive docs
